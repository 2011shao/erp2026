# uniadmin 框架开发计划

## 一、当前代码分析

### 现状
- 前端代码在 `app/` 目录
- 后端代码在 `admin/` 目录
- 已创建核心数据库Schema和用户管理云函数
- 代码结构不符合 uniadmin 框架规范

### 问题
1. 目录结构不符合 uniadmin 框架要求
2. 缺少 uniadmin 框架的核心功能模块
3. 缺少 uni-id 集成
4. 缺少菜单管理、权限管理等预置功能
5. 缺少 uniadmin UI 组件库

## 二、uniadmin 框架要求

### 核心结构
```
┌── uniCloud                            # 云端文件
│   │── cloudfunctions                  # 云函数相关文件
│   └── database                        # 数据库相关文件
├── common
│   │── admin-icons.css                 # admin-icons 图标样式
│   │── theme.scss                      # 换肤功能样式
│   │── uni.css                         # 公共样式
│   └── uni-icons.css                   # icon样式
├── components                          # 自定义组件
├── i18n                               # 国际化语言包目录
├── js_sdk                             # js sdk目录
├── pages                               # 页面
│   │── demo                            # demo相关的页面
│   │── error                           # 404页面
│   │── index                           # 首页
│   │── system                          # 系统管理
│   │   │── app                         # 应用管理
│   │   │── menu                         # 菜单管理
│   │   │── permission                   # 权限管理
│   │   │── role                         # 角色管理
│   │   │── safety                       # 安全审计
│   │   │── tag                          # 标签管理
│   │   └── user                         # 用户管理
│   └── uni-stat                         # uni统计
├── static                              # 静态资源
├── uni_modules                         # 插件模块
├── admin.config.js                     # 管理配置
├── App.vue                             # 应用入口
├── main.js                             # 主入口文件
├── manifest.json                       # 应用配置
└── pages.json                          # 页面路由配置
```

### 核心功能
1. **用户管理** - 基于 uni-id 的用户认证和权限管理
2. **角色管理** - 角色创建、权限分配
3. **菜单管理** - 动态菜单配置
4. **权限管理** - 基于角色的权限控制
5. **系统设置** - 系统配置管理
6. **数据统计** - uni统计集成
7. **插件生态** - 支持扩展插件

## 三、开发计划

### 阶段一：项目初始化
1. **创建 uniadmin 项目**
   - 使用 HBuilderX 创建 uniadmin 项目模板
   - 关联 uniCloud 服务空间
   - 配置基本信息

2. **目录结构搭建**
   - 按照 uniadmin 框架要求创建完整目录结构
   - 配置 `admin.config.js`
   - 配置 `manifest.json` 和 `pages.json`

### 阶段二：核心功能开发

1. **用户认证系统**
   - 集成 uni-id 认证
   - 实现登录、注册、密码修改功能
   - 配置用户角色权限

2. **系统管理模块**
   - 菜单管理
   - 角色管理
   - 权限管理
   - 用户管理

3. **业务功能模块**
   - 店铺管理
   - 商品管理
   - 库存管理
   - 销售管理
   - 财务管理
   - 报表分析

### 阶段三：数据库配置

1. **数据库Schema**
   - 创建 uni-id 相关数据表
   - 创建业务数据表
   - 配置数据权限

2. **云函数开发**
   - 开发核心云函数
   - 集成 uni-id 认证
   - 实现业务逻辑

### 阶段四：前端开发

1. **页面开发**
   - 系统管理页面
   - 业务功能页面
   - 报表页面

2. **组件开发**
   - 业务组件
   - 通用组件
   - 表单组件

3. **权限控制**
   - 页面权限
   - 按钮权限
   - 数据权限

### 阶段五：测试与优化

1. **功能测试**
   - 核心功能测试
   - 权限测试
   - 性能测试

2. **优化**
   - 性能优化
   - 安全优化
   - 用户体验优化

3. **部署**
   - 云函数部署
   - 前端部署
   - 系统上线

## 四、技术栈

### 前端
- **框架**: uni-app (Vue 3 + TypeScript)
- **UI组件**: uni-admin UI
- **状态管理**: Pinia
- **构建工具**: Vite

### 后端
- **云服务**: uniCloud
- **数据库**: uniCloud 云数据库
- **认证**: uni-id
- **云函数**: Node.js
