# 多门店 ERP 系统架构设计

## 技术架构

### 前端技术栈
- **框架**：React 18
- **语言**：TypeScript
- **UI 组件库**：Ant Design 5.x
- **状态管理**：Zustand
- **路由**：React Router v6
- **图表库**：Chart.js
- **构建工具**：Vite 5.x
- **样式**：Tailwind CSS

### 后端技术栈
- **框架**：Express.js
- **语言**：TypeScript
- **ORM**：Prisma 5.x
- **数据库**：SQLite（开发环境）
- **认证**：JWT
- **密码加密**：bcryptjs

## 核心模块设计

### 1. 路由模块
- 使用 React Router v6 实现前端路由
- 主要路由包括：首页、店铺管理、商品管理、库存管理、销售管理、财务管理、报表分析、用户管理

### 2. 状态管理模块
- 使用 Zustand 管理应用状态
- 主要状态包括：用户信息、店铺列表、商品列表、库存信息、订单信息、财务记录等

### 3. 数据模型

#### 用户模型
- id: 唯一标识符
- username: 用户名
- password: 密码（加密存储）
- role: 角色（admin/manager/staff）
- shopId: 所属店铺（admin 为 null）
- createdAt: 创建时间

#### 店铺模型
- id: 唯一标识符
- name: 店铺名称
- address: 地址
- phone: 电话
- manager: 负责人
- createdAt: 创建时间

#### 商品模型
- id: 唯一标识符
- name: 商品名称
- category: 分类
- brand: 品牌
- model: 型号
- price: 价格
- costPrice: 成本价
- stock: 库存
- shopId: 所属店铺
- createdAt: 创建时间

#### 库存日志模型
- id: 唯一标识符
- productId: 商品 ID
- quantity: 变动数量
- type: 变动类型（in/out）
- reason: 变动原因
- shopId: 所属店铺
- createdAt: 创建时间

#### 销售订单模型
- id: 唯一标识符
- shopId: 所属店铺
- totalAmount: 总金额
- status: 状态（pending/completed/cancelled）
- createdAt: 创建时间

#### 销售订单商品模型
- id: 唯一标识符
- orderId: 订单 ID
- productId: 商品 ID
- quantity: 数量
- price: 单价

#### 财务记录模型
- id: 唯一标识符
- shopId: 所属店铺
- amount: 金额
- type: 类型（income/expense）
- category: 分类
- description: 描述
- createdAt: 创建时间

## API 设计

### 认证相关 API
- POST /api/auth/login - 用户登录
- POST /api/auth/register - 用户注册
- GET /api/auth/me - 获取当前用户信息

### 店铺相关 API
- GET /api/shops - 获取店铺列表
- POST /api/shops - 创建店铺
- GET /api/shops/:id - 获取店铺详情
- PUT /api/shops/:id - 更新店铺
- DELETE /api/shops/:id - 删除店铺

### 商品相关 API
- GET /api/products - 获取商品列表
- POST /api/products - 创建商品
- GET /api/products/:id - 获取商品详情
- PUT /api/products/:id - 更新商品
- DELETE /api/products/:id - 删除商品

### 库存相关 API
- GET /api/inventory - 获取库存列表
- POST /api/inventory/logs - 添加库存变动记录
- GET /api/inventory/logs - 获取库存变动记录

### 销售相关 API
- GET /api/sales - 获取销售订单列表
- POST /api/sales - 创建销售订单
- GET /api/sales/:id - 获取销售订单详情
- PUT /api/sales/:id - 更新销售订单状态

### 财务相关 API
- GET /api/financial - 获取财务记录列表
- POST /api/financial - 创建财务记录
- DELETE /api/financial/:id - 删除财务记录

### 报表相关 API
- GET /api/reports/sales - 获取销售报表数据
- GET /api/reports/inventory - 获取库存报表数据
- GET /api/reports/financial - 获取财务报表数据
- GET /api/reports/shops - 获取店铺报表数据

## 部署架构

### 开发环境
- 前端：Vite 开发服务器（http://localhost:3000）
- 后端：Express 开发服务器（http://localhost:8000）
- 数据库：SQLite

### 生产环境
- 前端：静态文件部署（Nginx/Apache）
- 后端：Node.js 服务器（PM2 管理）
- 数据库：PostgreSQL/MySQL

## 安全考虑

### 前端安全
- 输入验证：使用 Ant Design Form 进行表单验证
- XSS 防护：React 内置 XSS 防护
- CSRF 防护：使用 JWT 认证

### 后端安全
- 认证授权：JWT 认证
- 密码加密：bcryptjs 加密存储
- 输入验证：使用 express-validator
- SQL 注入防护：使用 Prisma ORM
- CORS 配置：配置适当的 CORS 策略

## 性能优化

### 前端优化
- 代码分割：使用 React.lazy 和 Suspense
- 状态管理优化：合理使用 Zustand
- 渲染优化：使用 React.memo 和 useCallback
- 图片优化：使用适当的图片格式和大小

### 后端优化
- 数据库索引：为频繁查询的字段添加索引
- 缓存：使用 Redis 缓存热点数据
- 并发处理：使用 async/await 和 Promise
- 错误处理：统一的错误处理机制
