# 数据库迁移指南：从SQLite到PostgreSQL

## 概述

本文档详细说明了如何将多门店ERP系统从SQLite迁移到PostgreSQL数据库。

---

## 一、为什么选择PostgreSQL

### PostgreSQL的优势
- ✅ **企业级功能**：支持复杂查询、事务、存储过程等
- ✅ **高性能**：适合大规模数据和高并发访问
- ✅ **可扩展性**：支持主从复制、读写分离
- ✅ **数据完整性**：更强的约束和验证机制
- ✅ **开源免费**：活跃的社区和丰富的生态系统
- ✅ **地理信息系统**：内置PostGIS扩展，适合位置相关功能

### 适用场景
- 店铺数量超过20家
- 单表数据量超过100万条
- 需要高并发访问（>30并发用户）
- 需要复杂的数据查询和分析
- 需要数据备份和恢复策略

### 1. 系统要求

## 三、迁移步骤

### 方案一：使用Docker（推荐）

#### 步骤1：启动PostgreSQL容器

```bash
# 拉取PostgreSQL镜像
docker pull postgres:15

# 启动PostgreSQL容器
docker run -d \
  --name erp-postgres \
  -e POSTGRES_USER=erp_admin \
  -e POSTGRES_PASSWORD=erp_password123 \
  -e POSTGRES_DB=erp_system \
  -p 5432:5432 \
  -v postgres_data:/var/lib/postgresql/data \
  postgres:15
```

#### 步骤2：更新Prisma配置

编辑 `/workspace/api/prisma/schema.prisma`：

```prisma
datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}
```

#### 步骤3：创建环境变量文件

在 `/workspace/api/` 目录下创建 `.env` 文件：

```env
# PostgreSQL数据库连接
DATABASE_URL="postgresql://erp_admin:erp_password123@localhost:5432/erp_system?schema=public"

# JWT密钥（保持原有）
JWT_SECRET="your-secret-key-here"
```

#### 步骤4：运行数据库迁移

```bash
cd /workspace/api

# 生成Prisma客户端
npx prisma generate

# 创建迁移
npx prisma migrate dev --name init

# 运行seed脚本初始化数据
npx prisma db seed
```

## 四、数据迁移（从SQLite到PostgreSQL）

### 方法1：使用Prisma（推荐）

如果SQLite数据库中已有数据，可以使用以下方法迁移：

#### 步骤1：导出SQLite数据

```bash
cd /workspace/api

# 使用sqlite3命令导出数据
sqlite3 prisma/dev.db .dump > data_dump.sql
```

#### 步骤2：转换SQL格式

由于SQLite和PostgreSQL的SQL语法有差异，需要手动调整或使用工具转换。

#### 步骤3：导入到PostgreSQL

```bash
# 连接到PostgreSQL并导入
psql -h localhost -U erp_admin -d erp_system -f data_dump.sql
```

### 方法2：使用脚本迁移

创建一个迁移脚本，从SQLite读取数据并写入PostgreSQL：

```typescript
// api/scripts/migrate-data.ts
import { PrismaClient as PrismaClientSQLite } from '@prisma/client';
import { PrismaClient as PrismaClientPostgreSQL } from '@prisma/client/postgresql';

const sqlite = new PrismaClientSQLite({
  datasources: { db: { url: 'file:./prisma/dev.db' } }
});

const postgres = new PrismaClientPostgreSQL();

async function migrateData() {
  console.log('开始数据迁移...');
  
  // 迁移权限
  const permissions = await sqlite.permission.findMany();
  for (const permission of permissions) {
    await postgres.permission.create({ data: permission });
  }
  console.log('✓ 权限迁移完成');
  
  // 迁移角色
  const roles = await sqlite.role.findMany();
  for (const role of roles) {
    await postgres.role.create({ data: role });
  }
  console.log('✓ 角色迁移完成');
  
  // 继续迁移其他表...
  
  console.log('数据迁移完成！');
}

migrateData()
  .catch(console.error)
  .finally(async () => {
    await sqlite.$disconnect();
    await postgres.$disconnect();
  });
```

## 七、性能优化建议

### 1. PostgreSQL配置优化

编辑 `postgresql.conf`：

```conf
# 内存设置
shared_buffers = 1GB
effective_cache_size = 3GB
maintenance_work_mem = 256MB
work_mem = 16MB

# 检查点设置
checkpoint_completion_target = 0.9
wal_buffers = 16MB
default_statistics_target = 100

# 连接设置
max_connections = 100
```

### 2. 索引优化

确保常用查询字段有索引：

```sql
-- 为sales_order表添加复合索引
CREATE INDEX idx_sales_order_shop_date ON sales_order(shop_id, created_at);

-- 为product表添加索引
CREATE INDEX idx_product_shop_category ON product(shop_id, category);
CREATE INDEX idx_product_name ON product USING gin(to_tsvector('english', name));
```

### 3. 连接池配置

使用Prisma的连接池：

```prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

// 在代码中配置
const prisma = new PrismaClient({
  datasources: {
    db: {
      url: process.env.DATABASE_URL,
    },
  },
  // 连接池配置
  log: ['query', 'info', 'warn', 'error'],
});
```

## 八、监控和维护

### 1. 备份策略

```bash
# 每日自动备份
0 2 * * * pg_dump -U erp_admin erp_system > /backup/erp_$(date +\%Y\%m\%d).sql

# 压缩备份
gzip /backup/erp_$(date +\%Y\%m\%d).sql

# 保留最近30天的备份
find /backup -name "erp_*.sql.gz" -mtime +30 -delete
```

### 2. 慢查询日志

在 `postgresql.conf` 中启用：

```conf
log_min_duration_statement = 1000  # 记录超过1秒的查询
log_line_prefix = '%t [%p]: [%l-1] user=%u,db=%d,app=%a '
```

## 九、常见问题

### Q1: 迁移后查询变慢怎么办？
A: 运行 `ANALYZE` 更新统计信息，检查并优化慢查询。

### Q2: 如何处理连接超时？
A: 调整连接池大小，增加 `connect_timeout` 配置。

### Q3: PostgreSQL和SQLite的日期格式不同怎么办？
A: 使用Prisma的DateTime类型，它会自动处理格式转换。

### Q4: 可以同时使用两个数据库吗？
A: 可以，但不推荐生产环境使用。Prisma支持多数据源配置。
