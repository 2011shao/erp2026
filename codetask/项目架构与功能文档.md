# 多门店 ERP 系统架构与功能文档

## 1. 项目概览

多门店 ERP 系统是一个基于 React + TypeScript + Express + PostgreSQL 技术栈的企业资源规划系统，专为多门店管理设计。系统提供了完整的店铺管理、商品管理、库存管理、销售管理、财务管理和报表分析功能，支持角色权限控制和多用户管理。

### 核心功能：
- 店铺管理：添加、编辑、删除店铺信息
- 商品管理：商品的增删改查、分类管理、批量操作
- 库存管理：库存调整、库存预警、库存记录
- 销售管理：订单创建、订单状态管理、销售统计
- 财务管理：财务记录、收支分析、财务报表
- 报表分析：销售趋势、库存状态、财务分析
- 用户管理：用户创建、角色分配、权限控制
- 系统设置：角色管理、权限管理

## 2. 技术栈

### 前端技术栈：
- **框架**：React 18 + TypeScript
- **UI 组件库**：Ant Design 5.x
- **状态管理**：Zustand
- **路由**：React Router v6
- **HTTP 客户端**：Axios
- **样式**：Tailwind CSS
- **图表**：Ant Design Charts
- **工具库**：moment.js, xlsx

### 后端技术栈：
- **框架**：Express.js + TypeScript
- **ORM**：Prisma 5.x
- **数据库**：PostgreSQL
- **认证**：JWT
- **密码加密**：bcryptjs
- **中间件**：cors, express.json

## 4. 前端架构

### 4.1 组件结构

前端采用模块化的组件设计，主要分为以下几个部分：

1. **布局组件**：
   - 顶部导航栏：包含系统标题、全局搜索、用户信息
   - 侧边菜单栏：功能导航，支持折叠/展开
   - 内容区域：根据路由显示不同页面

2. **页面组件**：
   - HomePage：首页，显示统计数据和快捷操作
   - ShopPage：店铺管理
   - ProductPage：商品管理
   - InventoryPage：库存管理
   - SalesPage：销售管理
   - FinancialPage：财务管理
   - ReportPage：报表分析
   - UserPage：用户管理
   - RolePage：角色管理
   - PermissionPage：权限管理

3. **功能模块**：
   - 数据表格：支持排序、筛选、分页
   - 表单：支持验证、提交
   - 模态框：用于添加、编辑、删除操作
   - 统计卡片：显示关键指标
   - 图表：显示数据趋势

### 4.2 状态管理

使用 Zustand 进行状态管理，主要包括：

- **authStore**：管理用户认证状态、登录/登出、权限控制
- **dataCacheStore**：缓存 API 响应数据，提高性能

### 4.3 API 调用

使用 Axios 封装 API 调用，主要包括：

- **authApi**：认证相关接口
- **userApi**：用户管理接口
- **shopApi**：店铺管理接口
- **productApi**：商品管理接口
- **inventoryApi**：库存管理接口
- **salesApi**：销售管理接口
- **financialApi**：财务管理接口
- **reportApi**：报表分析接口

## 5. 后端架构

### 5.1 路由结构

后端采用 RESTful API 设计，主要路由包括：

- **/api/users**：用户管理
- **/api/shops**：店铺管理
- **/api/products**：商品管理
- **/api/inventory**：库存管理
- **/api/sales**：销售管理
- **/api/financial**：财务管理
- **/api/reports**：报表分析
- **/api/roles**：角色管理
- **/api/permissions**：权限管理

### 5.2 中间件

- **errorHandler**：统一错误处理
- **notFoundHandler**：404 处理
- **auth**：用户认证和权限验证

### 5.3 数据库模型

使用 Prisma ORM 定义数据库模型，主要包括：

- **User**：用户模型
- **Shop**：店铺模型
- **Product**：商品模型
- **InventoryLog**：库存变动记录
- **SalesOrder**：销售订单
- **SalesOrderItem**：销售订单详情
- **FinancialRecord**：财务记录
- **Permission**：权限表
- **Role**：角色表
- **RolePermission**：角色权限关联表
- **UserRole**：用户角色关联表

## 6. 数据库设计

### 6.1 数据库配置

系统使用 PostgreSQL 数据库，配置信息如下：

```
datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}
```

环境变量配置：
```
DATABASE_URL="postgresql://postgres:postgres@localhost:5432/shop_system?schema=public"
```

### 6.2 数据模型关系

- **User** 与 **Role**：多对多关系，通过 **UserRole** 表关联
- **Role** 与 **Permission**：多对多关系，通过 **RolePermission** 表关联
- **Shop** 与 **Product**：一对多关系
- **Shop** 与 **SalesOrder**：一对多关系
- **Shop** 与 **FinancialRecord**：一对多关系
- **Product** 与 **InventoryLog**：一对多关系
- **Product** 与 **SalesOrderItem**：一对多关系
- **SalesOrder** 与 **SalesOrderItem**：一对多关系

## 8. 认证与权限控制

### 8.1 认证流程

1. 用户登录，输入用户名和密码
2. 后端验证用户信息，生成 JWT token
3. 前端存储 token 到 localStorage
4. 后续请求携带 token 进行认证
5. 后端验证 token 有效性

### 8.2 权限控制

- **基于角色的权限控制 (RBAC)**：
  - 系统预定义角色：admin、manager、staff
  - 每个角色关联不同的权限
  - 用户可以分配多个角色
  - 权限分为：menu（菜单权限）、api（接口权限）、action（操作权限）

- **权限验证中间件**：
  - `authenticate`：验证用户是否登录
  - `authorize`：验证用户是否有特定权限

## 10. 系统配置

### 10.1 环境变量

#### 前端环境变量：
- `VITE_API_URL`：后端 API 地址，默认 `http://localhost:8000/api`

#### 后端环境变量：
- `DATABASE_URL`：PostgreSQL 数据库连接字符串
- `JWT_SECRET`：JWT 签名密钥
- `JWT_REFRESH_SECRET`：JWT 刷新令牌密钥
- `JWT_EXPIRES_IN`：JWT 过期时间，默认 `24h`
- `JWT_REFRESH_EXPIRES_IN`：JWT 刷新令牌过期时间，默认 `7d`
- `BACKEND_PORT`：后端服务端口，默认 `8000`

- **路由配置**：`/src/App.tsx` 中的 Routes 组件

- **懒加载**：路由懒加载
- **组件优化**：使用 React.memo、useCallback、useMemo
- **分页加载**：大量数据使用分页

### 11.2 后端优化

- **数据库索引**：为常用查询字段创建索引
- **批量操作**：支持批量添加、更新、删除
